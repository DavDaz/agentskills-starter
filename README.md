# agent-blueprint

Version control for AI agent skills. The source of truth lives here — deploy to wherever you need it.

## What's in here

| File | Purpose |
|------|---------|
| `skills/agents-init/SKILL.md` | The skill — restructures AGENTS.md after `/init` |
| `skills/agents-init/template.md` | Canonical AGENTS.md template with inline comments |
| `AGENTS-changes.md` | Design decisions and technical documentation |

## What agents-init does

Takes the raw `AGENTS.md` generated by `/init` (OpenCode or Claude Code) and restructures it into:

- **Skills reference table** — auto-detected from your `skills/` directory
- **Auto-invoke table** — extracted from skill `description` fields
- **200-line budget** — high-value content (skills, rules) in the first 200 lines; the rest after

## Deploy

Pick the tool you want to use this skill with and run the corresponding command from the repo root:

```bash
# OpenCode (recommended — already have skills here)
cp -r skills/agents-init ~/.agents/skills/

# Claude Code
cp -r skills/agents-init ~/.claude/skills/

# OpenCode native config
cp -r skills/agents-init ~/.config/opencode/skills/

# Symlink instead of copy — edits in the repo reflect immediately
ln -sf "$(pwd)/skills/agents-init" ~/.agents/skills/agents-init
ln -sf "$(pwd)/skills/agents-init" ~/.claude/skills/agents-init
```

## Usage

After deploying, open any project in OpenCode or Claude Code and run:

```
/agents-init
```

The skill reads your project's `AGENTS.md`, scans `skills/*/SKILL.md`, and rewrites the file
with proper structure. Nothing is deleted — content is reorganized.

## Frontmatter compatibility

The `SKILL.md` uses standard frontmatter (`name`, `description`) that works in both tools.
Claude Code-specific fields (`disable-model-invocation`, `allowed-tools`) are present for
Claude Code but silently ignored by OpenCode — no conflict.

| Field | OpenCode | Claude Code |
|-------|----------|-------------|
| `name` | ✅ required | ✅ used as slash command |
| `description` | ✅ required | ✅ used for auto-invocation |
| `disable-model-invocation` | ignored | ✅ prevents auto-invocation |
| `allowed-tools` | ignored | ✅ restricts tool access |
